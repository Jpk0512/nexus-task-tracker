Notes screen redesign — pick a layout

Research synthesis from Apple Notes, Obsidian, Bear, Notion, Craft, Logseq, and mail clients (Gmail/Superhuman). Pattern that wins: list is a constrained rail · editor/preview is the primary surface · panes are drag-resizable. Your current screen does the opposite — the list expands to fill leftover space while the preview is locked at 420px.

Apple Notes folders · list ~300px · editor fills Obsidian tree · editor primary · drag dividers Bear tags · compact list · big editor Gmail list capped · reading pane grows Notion single page (no dual pane)
Current problem this fixes: list grows infinitely wide (sparse rows, tags floating mid-void); preview stuck at fixed 420px. Option A caps the list and makes the editor the hero with a drag handle.

How it works

Two panes. Left = compact note list (default ~320px, drag 240–480px). Right = preview/edit (fills remaining width). Slim header: search + New only — long vault copy moves to Settings. Categories stay as section headers in the list.

Best default for us. Matches Apple Notes / Bear / Superhuman reading pane. Lowest learning curve; solves both of your complaints with one change.

Tradeoffs

  • Resizable preview — drag the gutter
  • List no longer wastes horizontal space
  • Keeps category grouping you already have
  • No folder tree (vault path still in subtitle)
  • Meta (backlinks) stays inside the editor footer

How it works

Browse mode: full-width list (still max-width constrained ~720px centered, not edge-to-edge sprawl) with richer rows. Open a note: transitions to split (list rail + large editor) or full editor with back. Similar to Things / some Linear detail flows.

Good if browsing is half the job. Two modes to maintain; slightly more complex state.

Tradeoffs

  • Browse without a skinny preview fighting for space
  • Editor only appears when you need it — huge when open
  • List can show denser metadata in browse mode
  • Context switch between browse and read
  • Harder to skim-while-reading side by side by default
