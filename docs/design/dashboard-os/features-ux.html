The experience layer (not the chrome)

The locked sidebar/home mockup is the frame. This page is the product: new capabilities, friendlier connections between surfaces, and concrete UX changes. Full prose: UX-FEATURE-DESIGN.md + decisions in §14.

Job of the app: get a thought from brain → trusted system → done or filed — and find it again without archaeology. Every feature below either cuts ceremony, cuts search cost, or connects two places.

New features

These are net-new product behaviors (or net-new compositions that feel like new features). Not “another settings page.”

NEW

1. Attention Graph + Do now

One ranking function: overdue → due today → in progress → blocked-needs-you → starter HITL → (single) capture-debt line. Home shows top 5 with reason chips. Click jumps to Focus with that item selected.

Why: stops shopping between My Tasks / Lens / Triage / Home cards.

NEW

2. Universal Capture (Dump)

Global capture defaults to Dump (uncommitted thought)—not Task, not Inbox. Modes: Dump | Todo | Task | Note | Update. Process later with promote chips.

Why: ADHD needs one door; Home bar today over-commits to tasks.

NEW · semantic

3. Needs you (sacred Inbox)

Attention queue only: mentions, waiting-on-you, starter questions, meeting actions assigned to you, MCP/agent failures needing you. Lives as Focus → Needs you. Never holds grocery lists.

Why: you said inbox = things that need your attention.

NEW · fix

4. Safe Promote menu

Shared menu on Capture, Todos, Meeting actions: → Todo | Task | Note. Creates target first, then archives source. Cancel never deletes the original (today’s todo promote deletes first).

MVP surfaces only — not everywhere yet.

NEW

5. Meeting Pipeline

Ingest → Summary + open actions (default) → bulk create tasks → file to project notebook → optional project update. Transcript is a secondary tab. Filters: needs filing / open actions.

Uses existing transcription API as step 1; summary via LM Studio → Gemini.

NEW · shape

6. Human Notebook Tree

Vault roots: Inbox · Projects · Areas · Resources. Each project = projects/{projectId}/ with _index, notes, decisions, meetings. Open in ZenNotes on any note.

Replaces agent folders (iter12/drafts) as the default UX.

NEW

7. Ask Vault

Chat over your notes with citation chips (must link real notes). Scope: whole vault or current project notebook. Same capability exposed to Claude via MCP later.

LM Studio first, Gemini fallback, always labeled.

NEW · composition

8. Project Place

Overview becomes a place: last initiative update, do-now-for-this-project, resource chips (Notes · Skills · MCPs · Secrets · Meetings), board peek. Not nine equal tabs.

UPGRADE

9. Initiative stream

Sticky “Drop an update…” composer + timeline (Linear-style). Health is a chip on the post. Feeds Home Project pulse.

Builds on existing health-updates API — new experience, not new DB invent.

NEW · shelf

10. Skills shelf (square tiles)

Small square cards, pin to project, shows on Project Place. Library scan stays underneath.

NEW · ops

11. MCP catalog

Top-level ops: status probe, tool count, used-by, link env from Secrets. Not buried in Settings.

NEW · ops

12. Secrets (Infisical)

Masked browse, import, inject into MCP/project. Never paste secrets into chat. API-keys become one source type under Secrets.

NEW · factory

13. Project Starter (in the loop)

Start from idea → workshop → seal emits same Project Place + notebook + board. Home “Continue starter” while in-flight. Thin entry now; full workshop after core graph.

NEW · tissue

14. Connection chips

Task row: project · note · meeting · blocked-by. Note rail: tasks · project · ZenNotes · Ask. One click instead of re-searching.

NEW · literacy

15. Filter language everywhere

Same filter grammar as Tasks on Focus, Notes, Capture, Meetings, Skills, MCPs, Secrets. Saved views where lists are long.

NEW · later

16. Capture Outline (WorkFlowy-lite)

Nested bullets, collapse, zoom, convert node → note/task. Sits as third Capture tab after Dump + Todos feel solid.

How the big ones work

Mechanics in plain language — enough to implement without guessing.

Attention Graph

  1. Collect candidatesYour open tasks, Needs you items, in-flight starter HITL, optional single capture-debt summary.
  2. ScoreOverdue > due today > in progress > blocked-needs-you > starter HITL > capture debt line.
  3. HomeShow top 5 with reason chips (“overdue · 3d”).
  4. ClickRoute Focus?segment=&id= — item selected, not a dead list.
  5. CompleteOptimistic remove from Do now (closure).

Safe promote

  1. User chooses target typeTodo / Task / Note (and project if required).
  2. Create destinationAPI create succeeds first.
  3. LinkWrite task↔note or meeting↔task ids when relevant.
  4. Archive sourceOnly after success. Cancel = nothing lost.

Meeting pipeline

  1. IngestPaste, file, or audio→transcription API.
  2. UnderstandSummary + decisions + actions (LM Studio → Gemini). User can edit.
  3. ActCheck actions → bulk tasks on project or todos.
  4. FileWrite note under projects/{id}/meetings/; mark filed.
  5. OptionalPost initiative update; open actions stay in Needs you until done.

Notebook + ZenNotes

  1. On project create / starter sealScaffold projects/{projectId}/ with _index.md.
  2. Browse in NexusTree by project name; id on disk.
  3. Deep editOpen in ZenNotes via URL/CLI on same vault root.
  4. AskEmbed chunks; answer with citations only to real paths.
